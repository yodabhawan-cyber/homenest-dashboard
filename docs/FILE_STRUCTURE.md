## Key Files (Must Understand These)

### 1. **proxy_v4.py** ⭐⭐⭐
**Purpose:** Main voice assistant brain. Routes queries to appropriate tier.

**Key Functions:**
- `classify_request(text)` - Determines which tier to use
- `call_openclaw(messages, profile)` - Routes to personal agent
- `switch_profile(text)` - Handles voice profile switching
- `handle_family_command(text)` - Routes to family manager
- `is_kid_safe_request(text)` - Content filtering

**Configuration:**
- Reads from: `.env`, `config/profiles.json`
- Writes to: `config/activity_log.json`

**Port:** 11434 (mimics Ollama API)

### 2. **dashboard/src/pages/Setup.jsx** ⭐⭐⭐
**Purpose:** Web-based setup wizard UI.

**Components:**
- `Setup` - Main wizard component
- `TestConnectionCard` - Connection test UI
- `FamilyMemberCard` - Family member form

**Flow:**
1. Step 1: API Keys
2. Step 2: Test Connections
3. Step 3: Add Family Members
4. Step 4: Complete Setup

**Calls:** `/api/setup/*` endpoints (setup-api.js)

### 3. **dashboard/setup-api.js** ⭐⭐⭐
**Purpose:** Setup wizard backend API.

**Endpoints:**
- `GET /api/setup/status` - Check if setup complete
- `POST /api/setup/test` - Test API connections
- `POST /api/setup/complete` - Generate all configs

**Actions on Setup Complete:**
- Writes `.env` file
- Creates agent workspaces
- Generates `profiles.json`
- Outputs `openclaw-agents-config.json`

**Port:** 3201

### 4. **config/profiles.json**
**Purpose:** Defines all family members and their settings.

**Format:**
```json
{
  "snehal": {
    "name": "Snehal",
    "pin": "111",
    "age": 35,
    "role": "parent",
    "voice": "piper",
    "personality": "professional",
    "agent_workspace": "agents/snehal/workspace"
  }
}
```

**Used By:** proxy_v4.py, dashboard

### 5. **agents/[name]/workspace/SOUL.md**
**Purpose:** Defines agent personality and behavior.

**Content:**
- Personality traits
- Communication style
- Access permissions
- Safety rules (for kids)

**Used By:** OpenClaw agent (when processing queries)

### 6. **dashboard/src/index.css**
**Purpose:** Global styles including "Digital Hearth" design system.

**CSS Variables:**
- `--accent-orange`: Primary brand color (#ff6b35)
- `--accent-purple`: Secondary brand color (#a855f7)
- `--bg-primary`, `--bg-secondary`: Background colors
- `--text-primary`, `--text-secondary`: Text colors

**Classes:**
- `.glass-card` - Main card style
- `.nav-item` - Navigation items
- `.stat-card` - Stat display cards

### 7. **dashboard/api-proxy.js**
**Purpose:** Proxies Home Assistant API requests from dashboard.

**Why:** Keeps HA token server-side (security).

**Endpoints:**
- `/api/states` - Get all HA entities
- `/api/services/*` - Call HA services
- `/api/camera_proxy/*` - Camera snapshots/streams

**Port:** 3200

### 8. **skills/family/family_manager.py**
**Purpose:** CLI tool for chores/rewards/bedtime logic.

**Commands:**
```bash
python3 family_manager.py chores today [name]
python3 family_manager.py chores complete [name] [chore]
python3 family_manager.py stars check [name]
python3 family_manager.py stars redeem [name] [reward]
python3 family_manager.py bedtime check [name]
```

**Used By:** proxy_v4.py (FAMILY tier)

### 9. **setup-wizard.py**
**Purpose:** Alternative CLI-based setup wizard.

**Usage:**
```bash
python3 setup-wizard.py
# Interactive prompts for family setup
```

**Generates:** Same outputs as web setup

**When to Use:** Advanced users, headless servers

### 10. **.env**
**Purpose:** Environment variables (API keys, URLs).

**Content:**
```bash
OPENAI_API_KEY=sk-...
HA_URL=http://homeassistant.local:8123
HA_TOKEN=eyJ...
OPENCLAW_TOKEN=...
ELEVENLABS_API_KEY=sk_...
```

**Security:** NEVER commit to git (.gitignore includes this)

## Directory Purposes

### `/config/`
**Purpose:** Runtime configuration (JSON files).
**Generated:** By setup wizard or manually edited.
**Shared:** Between voice proxy, dashboard, family manager.

### `/agents/`
**Purpose:** Personal AI agent workspaces.
**Generated:** By setup wizard (creates directory per family member).
**Used By:** OpenClaw Gateway (reads SOUL.md, MEMORY.md).

### `/dashboard/`
**Purpose:** Web-based control panel.
**Tech:** React + Vite + Tailwind CSS.
**Dev:** `npm run dev` (port 3100).
**Build:** `npm run build` → `dist/`.

### `/skills/`
**Purpose:** Modular voice assistant capabilities.
**Pattern:** Each skill has SKILL.md + implementation file(s).
**Example:** family/ skill handles chores, rewards, bedtime.

### `/docs/`
**Purpose:** Developer documentation.
**Audience:** Future developers, AI agents continuing work.

### `/logs/`
**Purpose:** Runtime logs (auto-created).
**Ignored:** In git (too large, changes frequently).

### `/launchd/` & `/systemd/`
**Purpose:** Service templates for auto-start on boot.
**Usage:** Copy to system directories, update INSTALL_PATH.

## File Naming Conventions

**Configuration:**
- `*.json` - Runtime config (profiles, chores, meals, etc.)
- `*-config.json` - Generated config (openclaw-agents-config.json)
- `.env` - Environment variables
- `setup-status.json` - Setup state

**Documentation:**
- `*.md` - Markdown documentation
- `UPPERCASE.md` - Important docs (README, INSTALL, etc.)
- `lowercase.md` - Component-specific docs

**Code:**
- `proxy_*.py` - Voice proxy versions
- `*-api.js` - Backend API servers
- `*_manager.py` - Management/utility scripts
- `*.jsx` - React components
- `*.css` - Stylesheets

**Agent Files:**
- `SOUL.md` - Agent personality (uppercase, important)
- `IDENTITY.md` - Agent identity
- `USER.md` - About the human
- `MEMORY.md` - Long-term memory
- `AGENTS.md` - Workspace instructions

## File Dependencies

**When you change...**

- `proxy_v4.py` → Restart voice proxy service
- `config/profiles.json` → Restart voice proxy (profile changes)
- `agents/*/workspace/SOUL.md` → Restart OpenClaw Gateway
- `dashboard/src/**` → Rebuild dashboard (`npm run build`)
- `.env` → Restart all services
- `config/chores.json` → No restart needed (hot-reloaded)

## Generated vs Static Files

**Generated (don't edit manually):**
- `.env` (setup wizard)
- `config/setup-status.json` (setup wizard)
- `config/activity_log.json` (voice proxy)
- `agents/*/workspace/MEMORY.md` (OpenClaw agents)
- `openclaw-agents-config.json` (setup wizard)
- `dashboard/dist/` (build output)
- `logs/*` (runtime logs)

**Static (edit as needed):**
- `proxy_v4.py`
- `dashboard/src/**`
- `config/chores.json` (can edit manually or via dashboard)
- `config/meals.json` (can edit manually or via dashboard)
- `config/rewards.json` (define rewards here)
- `agents/*/workspace/SOUL.md` (can edit to change personality)

## Important Paths for AI Agents

If you're an AI agent continuing development:

**To add a voice feature:**
1. Edit `proxy_v4.py`
2. Add patterns to `classify_request()`
3. Add handler function
4. Update docs

**To add a dashboard page:**
1. Create `dashboard/src/pages/NewPage.jsx`
2. Add route to `dashboard/src/App.jsx`
3. Add nav item if needed
4. Update docs

**To modify agent personality:**
1. Edit `agents/[name]/workspace/SOUL.md`
2. Restart OpenClaw Gateway
3. Test with voice commands

**To add a family feature:**
1. Define data structure in `config/` (new .json file)
2. Add logic to `skills/family/family_manager.py` if CLI needed
3. Add dashboard page if UI needed
4. Add voice patterns to proxy_v4.py if voice needed
5. Update docs

**To change design:**
1. Edit CSS variables in `dashboard/src/index.css`
2. Modify components in `dashboard/src/pages/`
3. Rebuild: `npm run build`
